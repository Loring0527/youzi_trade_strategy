## Context

现有 v2.x 策略基于情绪评分五阶段模型，由 AI 自动生成，存在以下问题：
- HOT 阈值70过敏感，牛市31%时间空仓
- 盈亏比0.78（亏损总额 > 盈利总额）
- WEAK_MID 从未触发
- 大量参数（阈值、乘数、窗口）缺乏原文依据

本次重写以 `docs/strategy_framework_wip.md` 为设计依据，以炒股养家原文为唯一策略来源。

聚宽平台约束：
- 日线级别回测，买入信号 T 日收盘生成，T+1 日开盘执行
- 无外部 import，`jqbacktest/main.py` 必须自包含
- 聚宽概念板块 API：`get_concept_stocks(concept_id)`，`get_concepts()`

## Goals / Non-Goals

**Goals:**
- 实现四层决策框架的完整回测版本
- 第一层：三态市场状态判断（赚钱效应/亏钱效应/恐慌效应）
- 第二层：热点概念识别 + 早鸟频率龙头识别 + 低吸信号
- 第三层：固定仓位规则（单笔30%，最多2只）
- 第四层：四优先级卖出管理
- 保持 strategy/ 本地模块与 main.py 逻辑同步

**Non-Goals:**
- 追涨（打板）信号（v2 考虑）
- 分钟级 intraday 择时
- 龙虎榜数据（付费接口）
- 恐慌效应下的超跌低吸（第一版先不实现）

## Decisions

### 决策1：废弃情绪评分，改用三态判断

**选择：** 直接用三个量化指标判断赚钱效应，不经过中间的评分层。

**理由：** 评分机制引入了加权参数（40%/30%/20%/10%），这些权重无原文依据，是噪声来源。三指标直接判断更简洁，每个指标都有明确的原文对应含义。

**备选方案：** 保留评分但重新校准权重 → 仍然引入无依据的参数，拒绝。

### 决策2：赚钱效应使用"全部通过"而非"加权评分"

**选择：** 三指标全部通过才操作，任一不通过立即空仓。

**理由：** 养家原文强调"宁可错过，不做错"，入场条件严格，出场条件敏感。加权方式会在某些指标很差时仍然入场，违背原则。

### 决策3：大盘指数定义为沪深平均

**选择：** `大盘指数日涨跌幅 = (上证指数涨跌幅 + 深证成指涨跌幅) / 2`

**理由：** 游资股票池横跨沪深两市，单用上证会因蓝筹权重过高低估中小盘恐慌程度。

### 决策4：只做低吸，不做追涨（v1）

**选择：** v1 只实现低吸信号，不实现打板追涨。

**理由：** 聚宽日线回测假设打板以涨停价成交，与实盘差异过大（实盘需要排队）。低吸以开盘价成交，回测假设更接近现实。

### 决策5：早鸟频率算法用60日滚动窗口

**选择：** 统计过去60个交易日的板块首发激活事件。

**理由：** 太短（如20日）样本不足，龙头地位不稳定；太长（如120日）可能包含已经过时的题材周期。60日约3个月，覆盖一个板块题材的完整生命周期。

## Risks / Trade-offs

- [早鸟频率计算量大] → 每日需要遍历所有概念板块，聚宽回测可能超时。缓解：只在赚钱效应确认时才计算，且批量获取数据。

- [低吸次日开盘价与实际买入价偏差] → 聚宽回测默认次日开盘成交，实盘中股票可能跳空高开或低开，成本差异较大。接受这个偏差，后续用分钟数据优化。

- [早鸟频率为0或样本过少] → 当某概念近60日激活次数很少时，早鸟频率缺乏统计意义。缓解：设置最小激活次数阈值（如 >= 3次），否则不买该板块。

- [聚宽概念板块数据延迟] → 概念板块成分股可能滞后于实际市场认知。接受，作为 v1 的已知局限。
